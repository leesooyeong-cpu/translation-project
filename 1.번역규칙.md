# 번역 규칙

## [1번규칙]

1. **말투**: 간결형
2. **형식**: 파일에 flowchart(순서도)는 유지하고 베트남어만 한글로 번역해줘

3. **효율성**: 작업과정에 대한 불필요한 설명은 생략하고 결과 보고만해줘. 파일이 3개 이상일 경우, 하나씩 번역을 완료할 때마다 진행 상황을 알려줘

4. 번역이 불가하면 최종 단계에서 메시지만 보내줘 (번역안해되 됨)

5. 최종 단계: 모든 파일이 생성되면 내 GitHub master 브랜치에 push 하고, [번역 완료 보고서] 메일이 내 워치로 오게 해줘.
## 번역 완료 및 예외 처리 규칙 (중요)

### [원칙] 사용자 개입(Y/N 입력)의 최소화
- 본 번역 프로세스는 **'완전 자동화(No-Touch)'** 및 **'손목 알림(갤럭시 워치) 최적화'**를 목표로 한다.
- 번역 과정 중 일부 단어의 번역이 불가능하거나 오류가 발생하더라도, 대기 상태(Y/N 팝업 또는 입력 요구)로 전환하여 프로세스를 중단시키지 않는다.

### [예외 발생 시 처리 기준]
6. **일부 번역 불가 단어 발생 시:**
   - 해당 단어는 무리하게 오역하지 않고 **원문(베트남어) 상태 그대로 유지**한다.
   - 프로세스를 멈추지 말고 즉시 파일명
   - 형태로 지정 폴더에 파일 저장을 강행한다.

7. **자동 Push 및 완료 처리:**
   - 번역 불가 건수가 있더라도 파일 저장이 완료되면 시스템 중단 없이 즉시 GitHub에 **자동 Push(Commit)**를 실행한다.

8. **최종 이메일 알림 양식 반영:**
   - 작업 완료 후 발송되는 Gmail 알림 본문에 조쉬님이 인지할 수 있도록 다음과 같이 예외 상황을 요약하여 포함한다.
   - *예시: "번역 완료 및 자동 Push 성공 (미번역/예외 항목: N건 존재 - 원문 유지됨)"*
9. 확정된 translation_dict.json을 기준으로 모든 파일의 베트남어를 한글로 일괄 치환(Replace)해서 번역해줘

### 일부단어 번역 규칙
## **용어 규칙**
9. 스탭프 or Dập Tự Động > 압착기
10. 그 일이 일어날때 > 발생시
11. 한살 > 연간 
12. 모든 시트는 맑은 고딕 적용
13. 한글을 확인할때 간결체로 변경해줘
14. 정해진 시간 없음 > 발생 시
15. Kiểm tra bằng mắt thường và thao tác hoạt động or 손과 눈은 정상이다 > 육안 및 동작 확인
16. Nghe > 청취
17. Không được hư hỏng và dịch chuyển (Y/N) or 손상시키거나 움직이지 마세요 (Y/N) > 손상여부 확인
18. 기계 > 설비

18. GitHub Push 시 윈도우 PowerShell 환경을 사용하므로, cat << 'EOF' 같은 리눅스 문법을 절대 사용하지 말고 윈도우 호환 명령어로 작성해줘
