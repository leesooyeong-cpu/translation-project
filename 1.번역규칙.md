# 번역 규칙

## [1번규칙]
1. **말투**: 간결형
2. **형식**: 파일에 flowchart(순서도)는 유지하고 베트남어만 한글로 번역해줘

3. **효율성**: 작업과정에 대한 불필요한 설명은 생략하고 결과 보고만해줘. 파일이 3개 이상일 경우, 하나씩 번역을 완료할 때마다 진행 상황을 알려줘

## 번역 완료 및 예외 처리 규칙 (중요)
### [원칙] 사용자 개입(Y/N 입력)의 최소화
### [예외 발생 시 처리 기준]
4. **일부 번역 불가 단어 발생 시:**
   - 해당 단어는 무리하게 오역하지 않고 **원문(베트남어) 상태 그대로 유지**한다.
   - 형태로 지정 폴더에 파일 저장을 강행한다.
5. **자동 Push 및 완료 처리:**
   - 번역 불가 건수가 있더라도 파일 저장이 완료되면 시스템 중단 없이 즉시 GitHub에 **자동 Push(Commit)**를 실행한다.
6. **최종 이메일 알림 양식 반영:**
   - 작업 완료 후 발송되는 Gmail 알림 본문에 인지할 수 있도록 다음과 같이 예외 상황을 요약하여 포함한다.
   - *예시: "번역 완료 및 자동 Push 성공 (미번역/예외 항목: N건 존재 - 원문 유지됨)"*

## [스크립트 및 구조 최적화]
7. 통합 스크립트(integrated_translator.py) 우선: 폴더별로 파일을 만들지 말고, 모든 폴더를 한 번에 처리할 수 있는 하나의 통합 엔진을 먼저 참고하여 사용한다.

8. 폴더 격리: 모든 파이썬(.py) 실행 도구는 프로젝트 루트가 아닌 scripts/ 폴더 안에 생성하고 관리한다.

9. 데이터 분리: 번역 사전은 json 형식으로, 작업 지침은 md 형식으로 유지하여 데이터와 로직을 철저히 분리한다.

### 일부단어 번역 규칙
## **용어 규칙**

10. 확정된 translation_dict.json을 기준으로 모든 파일의 베트남어를 한글로 일괄 치환(Replace)해서 번역해줘

11. 권한 승인 자동화 (Permissions): > - C:\hex_edit_project\ 폴더 및 하위 디렉토리에 대한 파일 읽기/쓰기 권한은 **항상 허용(Always Allow)**된 것으로 간주한다.
승인 팝업이 발생할 가능성이 있는 복잡한 Glob 패턴 대신, 직접적인 절대 경로를 사용하여 사용자 개입(Y/N) 없이 작업을 완수한다.
새로운 스크립트(.py) 생성 및 파일 수정 시 사용자의 별도 승인 없이 즉시 수행한다.
Edit 및 Create 도구 사용 시 '자동 승인' 모드를 기본으로 작동하도록 코드를 구성한다.

12. Git Push 최적화 (Windows):
Push 실패 시 git push --set-upstream origin master를 시도하여 업스트림 연결을 자동 설정한다.
core.autocrlf true 설정을 적용하여 LF/CRLF 관련 경고 및 중단을 방지한다.

13. GitHub Push 시 윈도우 PowerShell 환경을 사용하므로, cat << 'EOF' 같은 리눅스 문법을 절대 사용하지 말고 윈도우 호환 명령어로 작성해줘
